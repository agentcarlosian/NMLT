## Important failed approach

Do not repeat the monolithic proof that unfolds every generated equality and
runs one giant `simp_all`. It was stopped after more than two minutes. Lean
used about 2.5 GB and remained stable, but the approach is far too expensive
for normal CI. The attempted proof was fully reverted before `0a202b6`.

The successful `Grade` proof demonstrates the intended pattern: destructure
the two records, unfold only that generated equality, split its Boolean
branches, extract field equalities, and reconstruct native equality. Apply
that pattern in small lemmas.

The Aeneas array implementation is in:

```text
mechanization/lean/.lake/packages/aeneas/backends/lean/Aeneas/Std/Array/ArraySlice.lean
```

`core.slice.cmp.PartialEqSlice.eq_homo_spec` may help build a reusable array
lemma, although the generated code calls `core.array.equality.PartialEqArray.eq`.
Keep any bridge small and benchmark its Lean build time.
